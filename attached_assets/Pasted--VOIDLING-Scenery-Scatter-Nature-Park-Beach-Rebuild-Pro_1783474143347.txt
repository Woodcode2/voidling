# VOIDLING: Scenery Scatter, Nature + Park + Beach (Rebuild Prompt 5)
Fill the empty green and beach zones with clay scenery, scattered organically instead of placed on lots. This is the wave that closes the gap to the splash's lush island.

## ONE DESIGN CALL I MADE (flip it if you disagree)
Scenery is wired as EATABLE bonus food, using the exact same system the game already uses to let the void eat people and cars. Reason: if a tree were pure decoration, the void rolling over it and nothing happening would look broken. Making scenery devourable is consistent with "devour everything" and reuses code that already exists. Two safeguards: scenery does NOT count toward the percent-of-city-devoured win math (that stays tied to buildings and houses), and it uses the visual-bounds-only injection so eat-contact radius is size-correct. If you would rather scenery be pure non-eatable decoration for now, say the word and I flip Stage 4.

## GOAL
Scatter the nature, park, and beach cutouts across the forest, park, and beach zones so the island reads as lush and alive, drawn in the existing depth pass, eatable as low-value bonus food, with zero change to game balance.

## DO NOT TOUCH (leave fully intact)
- The percent-of-city-devoured / win calculation. Scenery must never be added to it.
- The eatable balance for people and cars, their values, spawn rates.
- The Prompt 3 building/house swap, the Prompt 4 people/vehicle swap, the Prompt 1 depth pass, the Prompt 2 juice.
- Lot generation and building/house placement. Scenery is separate and must not move a single building.
- Camera, roads, river, waterfall, void body rendering.
- Use the same injection approach as `clayCity.ts` and `clayLife.ts`: set only `spriteBounds`, never `spriteContactFrac`.

## STAGE 3: Scatter placement (this is the new pattern, prove it)
Create a new `clayScenery.ts` mirroring the others. Place scenery by SCATTER, not on lots:
- Nature: fill the forest zone densely with trees, bushes, rocks, logs, so it reads as a real forest. Sprinkle a lighter mix of trees, bushes, and flowers across the park and any general green/meadow areas so the whole island feels planted, not bare.
- Park: place the park props (playground slide, swing set, gazebo, benches, picnic table, soccer goal, pond, see-saw, ice cream cart, fountain, lamp, planter) spread across the park zone with breathing room between them.
- Beach: place the beach props (umbrella, towel, lifeguard tower, palm tree, rowboat, beach ball, sandcastle, surfboard, kiddie pool, pier, deck chairs) across the sand near the shore.
Gating rules, every placement must pass all of them:
- Inside the ISLAND polygon, AND inside its target zone.
- Not within road width of any road.
- Not on a building or house footprint, keep a small clearance.
- Not overlapping another scenery item, enforce a minimum spacing.
- Not in the river, lagoon, or open water. Beach items stay on sand.
Density caps (report the actual counts placed): forest trees and shrubs up to about 60, scattered greenery elsewhere up to about 50, park props about 12 to 18, beach props about 12 to 18. If frame rate drops hard, lower the caps and report it.
