## P0-A — Controls rewrite (the game lives or dies here)

The current controls feel laggy and the thumb covers the character. Replace the input system entirely:

- **Pointer Events only**: `pointerdown` / `pointermove` / `pointerup` with `setPointerCapture` on the canvas. Delete any mouse/touch duplicate handlers.
- CSS: `canvas { touch-action: none; }`, `html, body { overscroll-behavior: none; position: fixed; overflow: hidden; }`. Kill double-tap zoom, text selection, and pull-to-refresh completely.
- **Relative-drag control scheme (virtual joystick — the Hole.io/Agar standard):** on pointerdown anywhere, record that point as the anchor. Movement direction = vector from anchor to current pointer position. Speed = `clamp(distance / 110px, 0, 1) * maxSpeed`. On release, decelerate to zero over 200ms. The Voidling is **never** under the finger.
- While touching, draw a faint joystick hint: 15%-opacity anchor ring + a dot at the clamped pointer offset.
- Simulation on a fixed 60Hz timestep with interpolated rendering; clamp delta time to 50ms so backgrounding the tab never teleports anything.
- **Feel target:** direction changes register within one frame. If movement feels floaty after this, reduce smoothing until it feels like the void is glued to your intent.

## P0-B — Buttons and the death of beige

- Every tappable control is a real `<button>` element: min-height 64px, primary CTA full-width minus 24px side margins, generous padding, `:active` scale 0.96 over 80ms. Audit z-index — nothing may overlay a button and steal its taps. Tapping anywhere on the visible button must work.
- Grep the codebase for `#FDF6EC` and every cream/beige value and delete them all. New palette below is total.

## 1 — Visual overhaul: "Electric Pop"

- **UI screens (home, results, shop, boon cards):** background deep space violet `#14082B` with a sparse hand-drawn starfield; headings pure white in Fredoka; the PLAY button is hot coral `#FF3D68` with white text and a solid darker-coral bottom edge 6px tall (chunky 2.5D arcade button); secondary buttons electric yellow `#FFD23F` with `#14082B` text.
- **Arena:** saturated turquoise field `#1CC6AE`; paths `#FFD23F`; pond `#2D9CDB`; picnic blankets in coral/white check. No gradients on large surfaces — flat, vivid, confident color.
- **Sticker object style (this is the "crisp HD" look):** every object gets a saturated fill, a 3px solid white outline, a hard offset shadow (4px down, rgba(0,0,0,0.12), zero blur), and a simple two-tone highlight. Rendered at devicePixelRatio (cap 3) with no blurry scaling.
- Anti-patterns still banned: emoji as art, gradient soup, drop-shadow blur everywhere, more than 2 fonts, template-looking UI.

## 2 — Voidling glow-up (cuter)

- Eyes grow to ~30% of body height: white, big pupils, two shine dots each. Blink every 3–5 seconds, occasionally one eye first. Pupils track movement direction, or the nearest edible object when idle.
- A mouth: tiny contented smile at rest; opens wide when within 1.5 radii of something edible; visible chomp on every absorb.
- Cheek blush ovals. Idle breathing (scale 1.00–1.03, 2s cycle). Jelly wobble on sharp direction changes. Glow rim pulses brighter as the combo climbs.

## 5 — Rival voidlings: multiplayer feel, zero servers

Do **not** build networking. Build rivals behind a `RivalController` interface (`update(dt, world) → intents`) implemented today by `BotController`, so a future `NetController` can replace bots without touching game code.

- Spawn **5 rivals** per round: distinct body/glow colors, names drawn from pools and shown as `Name · CC` country tags (e.g., `Kai · JP`, `Luna · BR`, `Maks · PL`, `Ava · US`, `Rin · KR`), randomized each round.
- Starting sizes seeded around the player: two smaller, two similar, one bigger.
- **Bot brain** — a 3-state machine with 150–400ms reaction delays and wander noise so they feel human: `GRAZE` (seek the densest edible cluster), `HUNT` (chase any void or object ≥20% smaller within range), `FLEE` (run from any void ≥20% bigger within 4 radii). Bots absorb map objects and grow (they don't need a visible orbit ring — performance).
- **Eat rule, both directions:** overlap ≥60% of the smaller AND eater radius ≥ 1.15× the eaten → chomp. Eating a rival: they spiral in dramatically, you gain 50% of their mass, burst effect, banner "CHOMPED Luna · BR!", +500 × combo score.
- **Player gets eaten:** never game over. Ghost mode 2.5s (transparent, invulnerable), respawn at 65% of your mass with a wobbly "Ouch!". The clock remains the only true enemy.
- **Live leaderboard** top-left: six rows sorted by mass, You highlighted, updating in real time. Results screen adds final placement (#1–#6) and voids eaten; finishing #1 stamps a crown and pays bonus coins.
